## Purpose & scope boundary
Write `index.html` in the project root. It loads `words.json` (generated by generate.py) and renders a beautiful interactive word cloud using wordcloud2.js from CDN.

**In scope:** index.html only
**Out of scope:** generate.py, words.json generation

## Decisions in force

- Load `words.json` via `fetch('./words.json')` — must be served (or opened from same dir)
- Canvas-based rendering (wordcloud2.js default)
- Color scheme: varied, visually rich (not just one color)
- Dark background preferred
- Title: "My Claude Code History"
- Show word count / total words stats somewhere on the page

## Gotchas
- wordcloud2.js needs a canvas element with explicit width/height
- `fetch` won't work from file:// — the verification command uses `python3 generate.py && open index.html` — note this. Consider adding a small inline HTTP server option, or note in the HTML that a local server is needed. Best approach: include a `<script>` fallback that tries fetch, and if it fails (file:// protocol), show instructions to run `python3 -m http.server 8000`.
- Top 300 words is plenty for a good cloud — wordcloud2.js handles large lists fine
